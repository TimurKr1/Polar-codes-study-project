Код написан в качестве домашнего задания по дисциплине "Теория инфрмации и кодирования"
и не претендует на научную достоверность. Он позволяет собрать информацию о качестве работы
полярного кодирования при разных параметрах сообщения (длина, скорость) и разных параметрах
декодера (объем списка). Для простоты каждый раз передается сообщение, состоящее полностью из
нулей. Для каждой уникальной комбинации параметров строится зависимость отношения числа ошибочных
декодирований к общему числу декодирований (По умолчанию MAX_ITERS = 4000) от отношения мощности
сигнала к мощности шума.

Модуль polar_codes.py.
Функции для процесса полярного кодирования - декодирования сообщения.
Полярное кодирование происходит по алгоритму: исходное сообщение u делится на две части
a = u[:l//2] и b = u[l//2:], закодированное сообщение a = a xor b, b остается 
неизменным, закодированное полное сообщение получается конкатенацией новых a и b. Чтобы
закодировать все сообщение, нужно представить каждый бит сообщения в виде листьев дерева и
идти по дереву вверх для каждых пар a и b с шагом 2, сначала длина каждого
a и b равна 1, затем 2, так до момента, пока длины a и b не будут равны половине исходного 
сообщения.
Декодирование происходит при помощи алгоритма множественного принятия решений, сообщение 
декодируется по классическому алгоритму, но на каждом информационном бите строится два 
альтернативных пути, путь, который не удовлетворяет классическому алгоритму, штрафуется метрикой
пути, если путей накопилось больше, чем заданное кол-во, то остаются только те, у которых
метрика минимальная. Итоговое сообщение - с минимальной метрикой из оставшихся.

Модуль environment.py
Для создания сообщения используется файл rank.csv с отсортированной надежностью каналов из
стандарта 5G. На самые надежные каналы отправляются информационные биты, на остальные - замороженные.
Функция моделирования позволяет собрать информацию о FER (отношение кол-ва неверно декодированных
сообщений к кол-ву сообщений) в зависимости от заданного SNR (отношение мощности сигнала к 
мощности шума в дБ) для каждой заданной комбинации длины сообщения N, скорости сообщения R, длины
списка путей при декодировании L. По умолчанию установлено кол-во циклов кодирования - декодирования, 
равное 4000 для каждой комбинации (N, R, L, SNR). В процессе выводится отладочная информация
о том, какой процент комбинаций уже проверен, в файле simulation.py есть альтернативная версия
функции с более подробной отладочной информацией.
Функция построения графика позволяет построить на одном графике все полученные кривые FER vs SNR
и визуально сравнить качество передачи при разных параметрах сообщения. Для каждой длины сообщения
свой цвет, для каждой скорости свой стиль линии (4 штуки), для каждой длины списка свой стиль точек (4 штуки).
